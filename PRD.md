# 朋友圈分析交友H5产品需求文档

## 1. 产品概述

### 1.1 产品背景

在现代社交中，了解对方的兴趣爱好、生活方式和价值观是建立深度关系的基础。朋友圈作为个人生活的数字化展示窗口，包含了大量个人信息。本产品旨在通过分析用户上传的对方朋友圈截图，利用人工智能技术提取关键信息并生成分析报告，帮助用户更全面地了解对方，从而提高交友成功率。

### 1.2 产品目标

帮助用户通过分析对方朋友圈内容，更有效地与目标对象建立男女朋友关系。

### 1.3 目标用户

希望深入了解心仪对象并建立浪漫关系的成年人。

## 2. 核心功能

### 2.1 功能模块

| 模块名称 | 功能描述 |
|---------|--------|
| 图片上传 | 支持用户上传5张以上的朋友圈截图，支持批量上传和拖拽上传 |
| 图片预览 | 上传后可预览图片，支持删除和重新上传 |
| 模型分析 | 调用豆包大模型分析图片内容，提取个人信息和特征 |
| 分析报告 | 生成详细的个人分析报告，包括兴趣爱好、性格特点、生活方式等 |
| 交友建议 | 基于分析结果，提供针对性的交友建议和话题推荐 |
| 隐私保护 | 强调用户隐私保护，分析完成后自动删除上传的图片 |

### 2.2 页面流程

1. 用户进入H5页面
2. 上传至少5张朋友圈截图
3. 确认上传完成后提交分析
4. 等待模型分析完成（显示加载动画）
5. 查看分析报告和交友建议
6. 可选择保存报告或分享结果

## 3. 页面设计

### 3.1 设计风格

- 现代简约风格，以粉色和蓝色为主色调，营造浪漫温馨的氛围
- 响应式设计，适配各种移动设备屏幕
- 简洁清晰的视觉层次，突出核心功能
- 友好的交互反馈，增强用户体验

### 3.2 页面结构

1. **单页面应用**：集成上传、分析、报告展示功能于一体
   - 顶部：产品标题和简要说明
   - 中部：图片上传区域、上传进度提示、图片预览和管理
   - 底部：分析按钮、加载动画、分析结果展示、交友建议、分享/保存选项

### 3.3 关键交互

- 图片上传：支持拖拽上传、点击上传、批量选择
- 图片管理：支持预览、删除、重新排序
- 分析提交：点击按钮后显示加载状态
- 报告查看：滚动浏览详细报告，点击展开/收起部分内容

## 4. 技术实现

### 4.1 前端技术

- HTML5 + CSS3 + JavaScript
- 响应式布局（媒体查询）
- 图片上传和处理（File API）
- 表单验证
- 动画效果（CSS动画/JavaScript动画）

### 4.2 后端技术

- 服务器：Node.js/Express
- 图片处理：Sharp库
- API调用：Axios
- 临时存储：内存存储（分析完成后删除）

### 4.3 人工智能集成

- 模型：Doubao-Seed-1.6-vision
- API接口：使用火山引擎方舟平台的API
- 调用方式：参考官方文档 `https://www.volcengine.com/docs/82379/1399008`
- System Prompt：你是一个追求异性的高手，对于青春男女生的心理和外在表现，有非常强的洞察，也有一套很厉害的追求异性的技巧！擅长于输出简短但有效的分析和建议。

### 4.4 技术流程

1. 用户上传图片 → 前端验证数量和格式
2. 前端将图片发送到后端 → 后端接收并临时存储
3. 后端调用豆包大模型API → 传递图片数据
4. 模型分析图片内容 → 返回分析结果
5. 后端处理分析结果 → 生成结构化报告
6. 后端返回报告数据 → 前端展示给用户
7. 后端删除临时存储的图片 → 保护用户隐私

## 5. 数据分析

### 5.1 分析维度

| 分析维度 | 具体内容 |
|---------|--------|
| 兴趣爱好 | 从朋友圈内容中提取兴趣爱好，如旅游、美食、运动、阅读等 |
| 性格特点 | 分析发布内容的语气、频率、内容类型等，推断性格特征 |
| 生活方式 | 分析作息时间、社交活动、消费习惯等生活方式信息 |
| 价值观 | 从发布内容的观点、态度中提取价值观倾向 |
| 社交圈 | 分析互动频率、朋友类型等社交特征 |
| 情感状态 | 分析是否有恋爱对象、对恋爱的态度等 |

### 5.2 交友建议维度

| 建议维度 | 具体内容 |
|---------|--------|
| 话题推荐 | 基于兴趣爱好推荐聊天话题 |
| 约会建议 | 基于生活方式推荐约会场所和活动 |
| 沟通技巧 | 基于性格特点推荐沟通方式和技巧 |
| 关系进展 | 基于整体分析给出关系发展的建议 |

## 6. 隐私保护

### 6.1 数据处理原则

- 所有上传的图片仅用于分析目的，分析完成后立即删除
- 不存储用户上传的原始图片
- 不存储完整的分析结果，仅在会话期间展示
- 不将用户数据用于其他商业目的

### 6.2 隐私协议

在用户首次使用时，展示详细的隐私协议，明确告知数据处理方式，获得用户同意后才进行后续操作。

## 7. 风险控制

### 7.1 内容审核

- 对用户上传的图片进行初步审核，确保不包含违法违规内容
- 如发现违规内容，立即停止服务并提示用户

### 7.2 使用限制

- 限制单用户的分析频率，防止滥用
- 限制图片数量和大小，确保系统稳定性
